Networking

OSI Referans Modeli (Open Systems Interconnection - Açık Sistem Bağlantısı)

Bilgisayarların iletişim kurmasını standartlaştırmak ve çıkacak sorunların kaynağını bulmak için oluşturulmuştur. Toplamda 7 katmandan oluşur:
Physical (Fiziksel Katman)Veriler burada "bit" adını alır; 1 ve 0'lardan oluşur.
Data Link (Veri Bağı Katmanı) Veri paketlerinin hangi fiziksel cihaza gitmesi gerektiğini yönetir (Ör: Ethernet, Wi-Fi). Veriler burada "frame" adını alır.
Network (Ağ Katmanı) Verilerin ağda hangi yoldan gideceğini belirler (Ör: IP protokolü). Veriler burada "paket" adını alır.
Transport (Taşıma Katmanı) Veriyi doğru uygulamaya ve eksiksiz şekilde ulaştırır (TCP, UDP). Veriler burada "segment" adını alır. Yazılım katmanlarını donanım katmanlarına bağlar.
Session (Oturum Katmanı) Bilgisayarlar arasındaki konuşmayı başlatır, sürdürür ve bitirir.
Presentation (Sunum Katmanı) Veriyi bilgisayarın anlayacağı biçime çevirir.
Application (Uygulama Katmanı) Verilerle etkileşime girdiğimiz yerdir. Web tarayıcıları ve e-posta gibi uygulamalar bu katmanda çalışır. Veriler bu katmanda "veri"dir.

SDN (Software Defined Network)

Ağın çalışması için gerekli olan donanımların (switch, router vb.) nasıl çalışacağını yazılım üzerinden kontrol etmeye yarar.

OSI Katmanlarına Göre Protokoller ve Kavramlar

1. Application Layer (Uygulama Katmanı)


HTTP (Hypertext Transfer Protocol): Web sayfalarına erişimi sağlar.
FTP (File Transfer Protocol): Dosya transferi yapmayı sağlar.
SNMP (Simple Network Management Protocol): Ağdaki cihazlar arasında bilgi paylaşımını kolaylaştırır.

2. Presentation Layer (Sunum Katmanı)

Kodlama (Encoding): Verileri bilgisayarın anlayacağı dile çevirir. Örn: ASCII, UTF-8
Şifreleme (Encryption): Ağ üzerinden güvenli veri aktarımı sağlar.
SSL (Secure Sockets Layer): Baştan sona kadar şifreli iletişim. Her servis için (HTTP, POP3) ayrı port gerekir.
TLS (Transport Layer Security): Şifresiz başlar, sonra sertifikalama ile şifreleme başlar. Düz port kullanır.
Sıkıştırma (Compression): Veriyi küçültür, hızlı gönderim sağlar. Örn: GZip

3. Session Layer (Oturum Katmanı)

SOCKS: Proxy üzerinden paket göndermeyi sağlar.
NetBIOS: Ağ üzerindeki bilgisayarların veri alışverişini sağlar.
SIP (Session Initiation Protocol): Ağ üzerinden sesli veya görüntülü iletişim kurmaya yarar. En çok VoIP (Voice over IP) protokolünde kullanılır.

4. Transport Layer (Taşıma Katmanı)

TCP (Transmission Control Protocol): Verilerin sırasını korur, kaybolanları tekrar gönderir. Örn: HTTP, FTP, Gmail
UDP (User Datagram Protocol): Veri gider ama alındı mı diye kontrol etmez. Örn: YouTube, çevrimiçi oyunlar (hız önemli olduğunda kullanılır)
QUIC (Quick UDP Internet Connections): Google tarafından geliştirilmiştir. UDP hızı ile TCP güvenilirliğini birleştirir. Örn: HTTP/3 protokolünde kullanılır.

5. Network Layer (Ağ Katmanı)

IP (Internet Protocol): Verinin hangi adrese gideceğini belirler. IPv4 ve IPv6 türleri bulunur.
ICMP (Internet Control Message Protocol): Hata kontrolü ve bağlantı testi için kullanılır. (Ping komutunu çalıştırınca devreye girer.)
IPSec (Internet Protocol Security): IP'nin güvenli sürümüdür. VPN bağlantılarında kullanılır.

6. Data Link Layer (Veri Bağı Katmanı)

Ethernet: Kablolu ağ bağlantısı sağlar.
Wi-Fi (Wireless Fidelity): Kablosuz ağ bağlantısı sağlar.
NDP (Neighbor Discovery Protocol): IPv6 için komşu cihazları bulur.

7. Physical Layer (Fiziksel Katman)

Ethernet Physical Layer: Kablo üzerinden sinyalleri tanımlar.
Bluetooth Physical Layer: Veriyi radyo sinyaliyle iletir.
CAN Bus (Controller Area Network): Bilgisayarsız (PC'siz) cihazlarda, genellikle otomobillerde kullanılır.
